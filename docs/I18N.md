# Internationalization (i18n) Guide

## Overview

Boxcord supports multiple languages using **react-i18next**. English is the default language, and users can change their preferred language in Settings > Language.

## Supported Languages

- 🇬🇧 **English** (en) - Default
- 🇸🇪 **Svenska** (sv) - Swedish

## Configuration

The i18n configuration is located in `client/src/i18n/config.ts`. It:
- Loads translation files from `client/src/i18n/locales/`
- Saves the selected language to localStorage
- Syncs language changes across tabs
- Falls back to English if a translation is missing

## Usage in Components

### Basic Usage

```tsx
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();

  return (
    <div>
      <h1>{t('common.settings')}</h1>
      <button>{t('common.save')}</button>
    </div>
  );
}
```

### With Variables

```tsx
const { t } = useTranslation();

// In translation file:
// "welcome": "Welcome, {{name}}!"

return <div>{t('welcome', { name: userName })}</div>;
```

### Changing Language

```tsx
const { i18n } = useTranslation();

// Change to Swedish
i18n.changeLanguage('sv');

// Get current language
const currentLang = i18n.language;
```

## Translation Files

Translation files are JSON files located in `client/src/i18n/locales/`:

- `en.json` - English translations
- `sv.json` - Swedish translations

### File Structure

```json
{
  "common": {
    "save": "Save",
    "cancel": "Cancel"
  },
  "settings": {
    "title": "Settings",
    "profile": "Profile"
  }
}
```

## Adding New Translations

1. **Add to English file** (`en.json`):
   ```json
   {
     "myFeature": {
       "title": "My Feature",
       "description": "This is my feature"
     }
   }
   ```

2. **Add to Swedish file** (`sv.json`):
   ```json
   {
     "myFeature": {
       "title": "Min Funktion",
       "description": "Detta är min funktion"
     }
   }
   ```

3. **Use in component**:
   ```tsx
   const { t } = useTranslation();
   return <h1>{t('myFeature.title')}</h1>;
   ```

## Adding New Languages

1. **Create translation file**:
   ```bash
   touch client/src/i18n/locales/de.json  # German
   ```

2. **Add translations** (copy from `en.json` and translate)

3. **Import in config** (`client/src/i18n/config.ts`):
   ```ts
   import de from './locales/de.json';
   
   i18n.init({
     resources: {
       en: { translation: en },
       sv: { translation: sv },
       de: { translation: de }  // Add new language
     }
   });
   ```

4. **Add to language selector** (`client/src/components/settings/LanguageTab.tsx`):
   ```ts
   const LANGUAGES = [
     { code: 'en', name: 'English', flag: '🇬🇧' },
     { code: 'sv', name: 'Svenska', flag: '🇸🇪' },
     { code: 'de', name: 'Deutsch', flag: '🇩🇪' }  // Add new language
   ];
   ```

## Best Practices

### 1. Use Descriptive Keys
```tsx
// Good
t('messages.typeMessage')

// Bad
t('msg1')
```

### 2. Group Related Translations
```json
{
  "notifications": {
    "title": "Notifications",
    "enableDesktop": "Enable Desktop Notifications",
    "enableSound": "Enable Sound Effects"
  }
}
```

### 3. Avoid Hardcoded Strings
```tsx
// Bad
<button>Save</button>

// Good
<button>{t('common.save')}</button>
```

### 4. Handle Missing Translations
If a translation is missing, the key will be displayed. Always add translations to ALL language files.

### 5. Use Namespaces for Large Apps
For very large applications, consider splitting translations into namespaces:
```ts
t('common:save')      // from common namespace
t('settings:title')   // from settings namespace
```

## Testing Translations

1. **Switch language in Settings**:
   - Open Settings > Language
   - Select a different language
   - Verify all text updates

2. **Check for missing translations**:
   - If you see translation keys (like "common.save") instead of text, a translation is missing

3. **Test cross-tab sync**:
   - Open app in two tabs
   - Change language in one tab
   - Verify the other tab updates

## Language Persistence

The selected language is:
- Saved to `localStorage` under key `language`
- Automatically restored on page reload
- Synced across browser tabs
- Used immediately without page reload

## API

### useTranslation Hook

```tsx
const {
  t,           // Translation function
  i18n,        // i18n instance
  ready        // True when translations are loaded
} = useTranslation();
```

### Translation Function

```tsx
// Simple
t('common.save')

// With variables
t('welcome', { name: 'John' })

// With default value
t('missing.key', 'Default Value')

// With count (pluralization)
t('items', { count: 5 })
```

### i18n Instance

```tsx
// Change language
i18n.changeLanguage('sv');

// Get current language
const lang = i18n.language;

// Check if language exists
i18n.hasResourceBundle('sv', 'translation');

// Listen to language changes
i18n.on('languageChanged', (lng) => {
  console.log('Language changed to:', lng);
});
```

## Troubleshooting

### Translations not updating?
- Check if you imported `./i18n/config` in `main.tsx`
- Verify the translation exists in the JSON file
- Try clearing localStorage and reloading

### Getting translation keys instead of text?
- Translation is missing from the language file
- Add the translation to all language files

### Language not persisting?
- Check browser localStorage (should have `language` key)
- Verify localStorage is not disabled
- Check browser console for errors
